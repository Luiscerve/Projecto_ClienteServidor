1. ¿QUÉ ES ESTE PROYECTO?

1.1. Resumen

He creado una aplicación para gestionar una tienda. El sistema permite:

  • Crear categorías (Electrónica, Ropa, Alimentación...)
  • Crear productos dentro de esas categorías
  • Ver, modificar y borrar tanto categorías como productos

El proyecto tiene dos partes principales:

🖥️ SERVIDOR: Es un programa que está siempre encendido esperando peticiones. Guarda los datos en MySQL y responde cuando el cliente le pide información.

💻 CLIENTE: Es la aplicación que usa el usuario. Muestra menús por consola y se comunica con el servidor para hacer las operaciones.

1.2. ¿Por qué es útil?

Imagina que tienes una tienda y necesitas saber qué productos tienes, cuántos hay en stock y a qué precio están. Con este sistema puedes:

  • Añadir nuevos productos cuando lleguen al almacén
  • Ver todo el inventario
  • Modificar precios o cantidades
  • Borrar productos que ya no vendes
  • Organizar todo en categorías

1.3. Asignaturas que aplica

Asignatura ¿Qué he aplicado?
Programación de Servicios y Procesos La comunicación entre cliente y servidor usando HTTP (peticiones GET, POST, PUT, DELETE)
Acceso a Datos Guardar y leer datos de MySQL usando JPA (la forma fácil de usar bases de datos en Java)
Desarrollo de Interfaces Los menús de la aplicación de consola que ve el usuario

3. ¿CÓMO FUNCIONA EL SISTEMA?

3.1. Visión general

El sistema funciona así: el cliente envía peticiones al servidor, el servidor procesa la petición, consulta o modifica la base de datos, y devuelve una respuesta al cliente.

┌──────────────────┐ ┌──────────────────┐ ┌─────────────┐ │ USUARIO │ │ SERVIDOR │ │ MySQL │ │ (ve los menús) │ │ (Spring Boot) │ │ (guarda │ │ │ ── petición HTTP ──▶│ │ ── consulta SQL ──▶ │ datos) │ │ │ ◀── respuesta ──── │ │ ◀── resultado ──── │ │ └──────────────────┘ └──────────────────┘ └─────────────┘ │ │ │ │ └──────────────────────── El servidor es el "puente" ─────────────────────────────┘

3.2. Ejemplo: Crear un producto

Cuando el usuario quiere crear un producto nuevo, esto es lo que pasa:

  1. El usuario selecciona "Crear producto" en el menú del cliente
  2. El usuario escribe el nombre, precio, stock y categoría
  3. El cliente crea un objeto Producto con esos datos
  4. El cliente convierte el objeto a JSON (texto)
  5. El cliente envía una petición POST al servidor con ese JSON
  6. El servidor recibe la petición en el Controller
  7. El Controller llama al Service para procesar los datos
  8. El Service valida que todo esté bien y llama al Repository
  9. El Repository guarda el producto en MySQL
  10. El servidor devuelve el producto creado (con su ID nuevo) al cliente
  11. El cliente muestra por pantalla: "Producto creado con éxito"

4.1. Las tablas

La base de datos se llama tienda_db y tiene 2 tablas:

Tabla: categorias

Campo Tipo ¿Para qué sirve?

Campo Tipo ¿Para qué sirve?

Esto se llama relación 1 a N (uno a muchos):

4.3. Configuración

La conexión a MySQL se configura en el archivo application.properties:

# Puerto del servidor (donde escucha peticiones) server.port=12345 # Conexión a MySQL spring.datasource.url=jdbc:mysql://localhost:3306/tienda_db spring.datasource.username=root spring.datasource.password= # JPA crea las tablas automáticamente spring.jpa.hibernate.ddl-auto=update
¿Qué hace "ddl-auto=update"?
Cuando arrancas el servidor, Hibernate (JPA) mira tus clases Java (Categoria.java, Producto.java) y crea las tablas automáticamente en MySQL si no existen. Si ya existen, las actualiza si hace falta.

¡No tienes que escribir CREATE TABLE a mano!

4.4. Crear la base de datos

Lo único que tienes que hacer en MySQL es crear la base de datos:

CREATE DATABASE tienda_db;

Las tablas se crean solas cuando arrancas el servidor.

5. EL SERVIDOR (API REST)

5.1. ¿Qué es una API REST?

Una API REST es una forma de que dos programas se comuniquen por Internet usando peticiones HTTP (como un navegador web, pero sin interfaz gráfica).

Los tipos de petición más comunes son:

Método HTTP ¿Para qué se usa? Ejemplo
GET Obtener datos (leer) GET /api/productos → devuelve todos los productos
POST Crear algo nuevo POST /api/productos → crea un producto nuevo
PUT Modificar algo existente PUT /api/productos/5 → modifica el producto con id=5
DELETE Borrar algo DELETE /api/productos/5 → borra el producto con id=5

5.2. Endpoints del servidor

Un endpoint es una "dirección" del servidor que hace algo concreto. Mi servidor tiene 10 endpoints:

Endpoints de Categorías

Método URL ¿Qué hace?
GET /api/categorias Devuelve todas las categorías
GET /api/categorias/{id} Devuelve una categoría por su ID
POST /api/categorias Crea una categoría nueva
PUT /api/categorias/{id} Modifica una categoría
DELETE /api/categorias/{id} Borra una categoría

Endpoints de Productos

Método URL ¿Qué hace?
GET /api/productos Devuelve todos los productos
GET /api/productos/{id} Devuelve un producto por su ID
GET /api/productos/categoria/{id} Devuelve los productos de una categoría
POST /api/productos Crea un producto nuevo
PUT /api/productos/{id} Modifica un producto
DELETE /api/productos/{id} Borra un producto

5.3. Código del Controller

El CategoriaController recibe las peticiones HTTP. Aquí está el código simplificado:

@RestController @RequestMapping("/api/categorias") public class CategoriaController { @Autowired private CategoriaService categoriaService; // GET /api/categorias - Obtener todas @GetMapping public List<Categoria> obtenerTodas() { return categoriaService.obtenerTodas(); } // GET /api/categorias/5 - Obtener una por ID @GetMapping("/{id}") public Categoria obtenerPorId(@PathVariable Long id) { return categoriaService.obtenerPorId(id); } // POST /api/categorias - Crear nueva @PostMapping public Categoria crear(@RequestBody Categoria categoria) { return categoriaService.crear(categoria); } // PUT /api/categorias/5 - Actualizar @PutMapping("/{id}") public Categoria actualizar(@PathVariable Long id, @RequestBody Categoria categoria) { return categoriaService.actualizar(id, categoria); } // DELETE /api/categorias/5 - Eliminar @DeleteMapping("/{id}") public void eliminar(@PathVariable Long id) { categoriaService.eliminar(id); } }
¿Qué significan las anotaciones?
  • @RestController: Esta clase responde a peticiones HTTP
  • @RequestMapping("/api/categorias"): Todos los métodos empiezan con esa URL
  • @GetMapping: Responde a peticiones GET
  • @PostMapping: Responde a peticiones POST
  • @PathVariable: Coge el {id} de la URL
  • @RequestBody: Coge los datos JSON del cuerpo de la petición
  • @Autowired: Spring inyecta automáticamente el Service

5.4. Código del Service

El CategoriaService contiene la lógica de negocio:

@Service public class CategoriaService { @Autowired private CategoriaRepository categoriaRepository; public List<Categoria> obtenerTodas() { return categoriaRepository.findAll(); } public Categoria obtenerPorId(Long id) { return categoriaRepository.findById(id) .orElseThrow(() -> new ResourceNotFoundException("Categoría no encontrada")); } public Categoria crear(Categoria categoria) { return categoriaRepository.save(categoria); } public Categoria actualizar(Long id, Categoria datos) { Categoria categoria = obtenerPorId(id); categoria.setNombre(datos.getNombre()); categoria.setDescripcion(datos.getDescripcion()); return categoriaRepository.save(categoria); } public void eliminar(Long id) { Categoria categoria = obtenerPorId(id); categoriaRepository.delete(categoria); } }

5.5. Código del Repository

El Repository es muy simple porque Spring Data JPA hace casi todo:

@Repository public interface CategoriaRepository extends JpaRepository<Categoria, Long> { // No hace falta escribir nada más // JpaRepository ya tiene: findAll(), findById(), save(), delete()... }
¿Por qué está vacío?
JpaRepository ya incluye los métodos básicos (CRUD). Solo tenemos que decirle qué entidad usa (Categoria) y qué tipo tiene el ID (Long). Spring hace el resto.

5.6. Código de la Entidad

La entidad Categoria representa la tabla en la base de datos:

@Entity @Table(name = "categorias") public class Categoria { @Id @GeneratedValue(strategy = GenerationType.IDENTITY) private Long id; @Column(nullable = false, length = 100) private String nombre; @Column(length = 255) private String descripcion; // Constructores, getters y setters... }
¿Qué significan las anotaciones?
  • @Entity: Esta clase se guarda en la base de datos
  • @Table(name = "categorias"): El nombre de la tabla en MySQL
  • @Id: Este campo es la clave primaria
  • @GeneratedValue: El ID se genera automáticamente (1, 2, 3...)
  • @Column: Configura cómo será la columna en la tabla

6.4. Código del ApiClient

El ApiClient es la clase que se comunica con el servidor. Usa HttpURLConnection (incluido en Java, no necesita librerías externas):

public class ApiClient { private static final String BASE_URL = "http://localhost:12345/api"; private static final Gson gson = new Gson(); // Obtener todas las categorías public static List<Categoria> obtenerCategorias() throws Exception { String respuesta = hacerPeticionGET(BASE_URL + "/categorias"); Type tipo = new TypeToken<List<Categoria>>(){}.getType(); return gson.fromJson(respuesta, tipo); } // Crear una categoría public static Categoria crearCategoria(Categoria categoria) throws Exception { String json = gson.toJson(categoria); String respuesta = hacerPeticionPOST(BASE_URL + "/categorias", json); return gson.fromJson(respuesta, Categoria.class); } // Método para hacer peticiones GET private static String hacerPeticionGET(String url) throws Exception { HttpURLConnection conexion = (HttpURLConnection) new URL(url).openConnection(); conexion.setRequestMethod("GET"); return leerRespuesta(conexion); } // Método para hacer peticiones POST private static String hacerPeticionPOST(String url, String json) throws Exception { HttpURLConnection conexion = (HttpURLConnection) new URL(url).openConnection(); conexion.setRequestMethod("POST"); conexion.setRequestProperty("Content-Type", "application/json"); conexion.setDoOutput(true); // Enviar el JSON conexion.getOutputStream().write(json.getBytes("UTF-8")); return leerRespuesta(conexion); } }
¿Cómo funciona?
  1. Crea una conexión HTTP a la URL del servidor
  2. Configura el método (GET, POST, PUT, DELETE)
  3. Si es POST/PUT, envía los datos en formato JSON
  4. Lee la respuesta del servidor
  5. Convierte el JSON de vuelta a objetos Java con GSON

7.1. ¿Cómo he probado el proyecto?

Archivo Nº de pruebas ¿Qué prueba?

7.2. Ejemplo de prueba

Esta prueba comprueba que se puede crear una categoría correctamente:

@SpringBootTest class CategoriaServiceTest { @Autowired private CategoriaService categoriaService; @Autowired private CategoriaRepository categoriaRepository; @AfterEach void limpiarDatos() { // Borra los datos de prueba después de cada test categoriaRepository.deleteAll(); } @Test void testCrearCategoria() { // 1. Crear una categoría de prueba Categoria categoria = new Categoria(); categoria.setNombre("Test Categoria"); categoria.setDescripcion("Descripción de prueba"); // 2. Guardarla en la base de datos Categoria resultado = categoriaService.crear(categoria); // 3. Comprobar que se ha guardado bien assertNotNull(resultado.getId()); // Tiene ID assertEquals("Test Categoria", resultado.getNombre()); // El nombre es correcto } }
¿Qué hace @SpringBootTest?
Arranca toda la aplicación Spring Boot para que la prueba pueda usar los servicios y la base de datos real. Es una prueba de integración.
¿Qué hace @AfterEach?
Después de cada prueba, borra todos los datos que se han creado. Así cada prueba empieza con la base de datos limpia.
